# Test plan

- **Caso feliz**:
  - Confirmar que existe un diseño aprobado.
  - El plan incluye header obligatorio (Goal/Architecture/Tech Stack).
  - Las tareas están divididas en acciones pequeñas (2-5 min) con:
    - Files (rutas exactas)
    - Steps (pasos concretos)
    - Verify (comando/expected o checklist)
  - Se guarda en `docs/plans/YYYY-MM-DD-<feature-name>.md`.

- **Errores**:
  - Si no hay diseño: debe pedir usar primero `@skills:idear-diseno`.
  - Si faltan rutas exactas: debe pedir leer/abrir archivos antes de inventarlas.

- **Regresión**:
  - Repetir un plan de otro tema y confirmar que todas las tareas tienen Verify.
